| 項目 | 定義 |
|------|------|
| これは何か | ライダーの存在証明 — 足跡が残る地図 |

## 1. タイポグラフィ

### 1-1. フォントスタック

React Native のデフォルトシステムフォント（iOS: San Francisco / Hiragino Sans、Android: Noto Sans CJK JP）を使用する。カスタムフォントは導入しない。

**原則:**
- 和文フォントが必ずシステムのフォールバックチェーンに含まれることを前提とする
- `fontFamily` を明示指定しない（プラットフォームのデフォルトに委ねる）
- 英数字と和文が混在する場面（スポット名、バイク車種名）が頻出するため、混植の自然さを最優先する

### 1-2. サイズ・行間・ウェイト

| 要素 | fontSize | fontWeight | lineHeight 倍率 | 用途 |
|------|----------|------------|-----------------|------|
| 画面タイトル | 32 | `'700'` | 1.3 | 各画面の見出し |
| セクション見出し | 24 | `'700'` | 1.35 | カード内見出し |
| 小見出し | 20 | `'600'` | 1.4 | サブセクション |
| 本文 | 16 | `'400'` | 1.7 | 説明テキスト、口コミ |
| 補助テキスト | 14 | `'400'` | 1.6 | キャプション、タイムスタンプ |
| ラベル | 13 | `'500'` | 1.4 | フォーム・バッジ・タブ |
| 極小 | 12 | `'400'` | 1.45 | 注釈・法的テキスト |

**lineHeight の計算:** `lineHeight = fontSize × 倍率`（React Native は px 絶対値で指定）

### 1-3. 和文テキストの禁則

| ルール | 説明 |
|--------|------|
| 本文の行間は 1.6〜1.8 を死守 | 和文は字面が大きく、行間が詰まると圧迫感が出る。1.5 未満は禁止 |
| 見出しの行間は 1.3〜1.4 | 見出しは本文より詰めてよい。ただし 1.2 未満は禁止 |
| `letterSpacing` は本文に使わない | 和文に letterSpacing を入れると文字間が不均一に崩れる。0 固定 |
| 見出しのみ微量許可 | 見出しに限り `letterSpacing: 0.5` まで許可（ブランド演出時のみ） |
| 長文は必ず折り返しテスト | 3行以上に折り返されるテキストは実機で読みやすさを確認する |

### 1-4. 混植（和欧混在）ルール

Moto-Logos では以下のテキストが頻繁に混植になる:
- スポット名: 「渋谷マークシティ バイク駐車場」
- バイク車種名: 「CBR650R」「XSR900」「PCX150」
- 料金表記: 「1時間 ¥200」「30分無料」
- UI ラベル: 「Moto-Logos」ブランド名

| ルール | 説明 |
|--------|------|
| 英数字が和文と並ぶ時はスペースを入れない | React Native のシステムフォントが自動調整する。手動スペースは過剰 |
| 英語のみのラベルは避ける | 「Settings」ではなく「設定」。ブランド名「Moto-Logos」は例外 |
| 車種名・固有名詞はそのまま | CBR650R、JMPSA 等は英字のまま表示。無理に和訳しない |
| 数字 + 単位は半角 | 「¥200」「52pt」「50m」。全角数字は使わない |

## 2. カラーシステム

### 2-1. パレット（`src/constants/theme.ts` が正）

| トークン | 値 | 用途 |
|---------|-----|------|
| `bg` | `#000000` | メイン背景 |
| `sheet` | `#1C1C1E` | ボトムシート・モーダル |
| `surface` | `#1C1C1E` | 設定・通知画面の背景 |
| `card` | `#1C1C1E` | 一次カード |
| `cardElevated` | `#2C2C2E` | 二次カード（SpotDetail等） |
| `text` | `#F2F2F7` | 主テキスト |
| `sub` | `#8E8E93` | 副テキスト |
| `accent` | `#FF6B00` | ブランドオレンジ（CTA・強調） |
| `border` | `rgba(255,255,255,0.10)` | 区切り線 |
| `hairline` | `rgba(255,255,255,0.08)` | 微細な区切り |

### 2-2. セマンティックカラー

| 用途 | トークン | 値 |
|------|---------|-----|
| 成功・停められた | `green` | `#30D158` |
| 危険・満車 | `red` | `#FF453A` |
| 警告・料金違い | `yellow` | `#FFD60A` |
| 情報・リンク | `blue` | `#0A84FF` |
| ユーザー投稿 | `purple` | `#BF5AF2` |

### 2-3. カラールール

| ルール | 説明 |
|--------|------|
| 色だけで意味を伝えない | アイコン・ラベルを必ず併用する（駐車温度もアイコン+色） |
| accent は CTA のみ | ブランドオレンジの乱用禁止。画面内で accent は原則1箇所 |
| 背景の階層を守る | `bg` → `surface/sheet` → `card` → `cardElevated` の順。逆転させない |
| ダーク背景での白テキスト | `text`（#F2F2F7）はコントラスト比 15:1 以上。`sub`（#8E8E93）は補助情報限定 |

## 3. スペーシング

### 3-1. スケール（`Spacing` トークン）

| トークン | 値 | 用途 |
|---------|-----|------|
| `xs` | 4 | アイコンとラベルの間 |
| `sm` | 8 | カード内パディング（最小） |
| `md` | 16 | 標準パディング・マージン |
| `lg` | 24 | セクション間 |
| `xl` | 32 | 画面端マージン（左右） |
| `xxl` | 48 | 大セクション間 |

### 3-2. スペーシングルール

| ルール | 説明 |
|--------|------|
| マジックナンバー禁止 | `padding: 13` のような任意値は使わない。必ず `Spacing` トークンを使う |
| 画面の左右余白は `Spacing.md`（16）以上 | 片手持ちでの視認性確保 |
| タップ領域間のギャップは `Spacing.sm`（8）以上 | 隣接ボタンの誤タップ防止 |
| リストアイテム間は `Spacing.sm`〜`Spacing.md` | 密度と可読性のバランス |
| 和文本文の段落間は `Spacing.md`（16）以上 | 行間 1.7 と合わせて呼吸感を確保 |

## 4. タッチターゲットとグローブ対応

### 4-1. サイズ基準（`ButtonSize` トークン）

| トークン | 値 | 用途 |
|---------|-----|------|
| `giant` | 80pt | 特大アクション（今後用） |
| `large` | 64pt | 主要アクション（案内開始・停めた）|
| `normal` | 52pt | 標準ボタン・タブ |

### 4-2. タッチルール

| ルール | 説明 |
|--------|------|
| 最小タップ領域は 52×52pt | Apple HIG 44pt より大きい。グローブを想定 |
| 主要 CTA は 64pt 以上 | 「案内開始」「ここに停めた」等の走行前後アクション |
| `hitSlop` を活用する | 視覚的に小さくても、タップ領域は 52pt を確保する |
| ボタン内テキストは 1行に収める | 日本語ラベルが折り返す場合はラベルを短縮する |
| 片手（右手親指）で届く範囲に CTA を配置 | 画面下半分。上部は情報表示に使う |

## 5. コンポーネント密度

### 5-1. カード

| ルール | 説明 |
|--------|------|
| 内側パディングは `Spacing.md`（16）以上 | カード内テキストが端に張り付かない |
| カード間マージンは `Spacing.sm`（8）以上 | 視覚的分離を確保 |
| 情報密度は「一目で3項目まで」 | スポットカード: 名前・距離・温度。それ以上は展開後に |
| カード内の区切りは `hairline` で最小限 | 罫線だらけにしない。余白で区切る |

### 5-2. リスト（NearbySpotsList 等）

| ルール | 説明 |
|--------|------|
| 1行の情報量は「名前 + 1〜2個の属性」 | 距離、温度、台数のうち最大2つ |
| 行の高さは 56pt 以上 | グローブタップ可能な最小高さ |
| セパレーターは `hairline` | 太いボーダーは使わない |
| スクロール方向は縦のみ | 横スクロールリストは使わない（片手操作を考慮） |

### 5-3. フォーム（レポート・設定）

| ルール | 説明 |
|--------|------|
| ラベルはフィールドの上に配置 | 左配置（inline label）は和文では窮屈になる |
| 入力フィールドの高さは 48pt 以上 | 日本語 IME のプレビュー行を考慮 |
| フィールド間のギャップは `Spacing.md`（16） | フォーム全体が詰まりすぎない |
| プレースホルダーは具体例を示す | 「入力してください」ではなく「渋谷駅」「CBR650R」|
| 必須/任意の表示 | 任意項目に「（任意）」と明記。必須がデフォルト |

### 5-4. ボトムシート（SpotDetailSheet）

| ルール | 説明 |
|--------|------|
| 上部はスクロール可能な情報ゾーン | スポット名・料金・口コミ等 |
| 下部は固定のアクションゾーン | 「案内開始」「足跡を残す」等の CTA |
| アクションゾーンの CTA は最大2つ | 選択肢が多すぎると迷う |
| シート背景は `sheet`（#1C1C1E） | 地図との視覚的分離 |

## 7. アニメーションとモーション

| ルール | 説明 |
|--------|------|
| 駐車温度の脈動は `react-native-reanimated` | ネイティブドライバーで60fps確保 |
| 画面遷移は OS 標準に従う | カスタムトランジション不要。iOS: push/pop、Android: fade |
| ローディングは控えめ | 全画面スピナー禁止。インライン or スケルトンで |
| アニメーション時間は 200〜400ms | それ以上は「遅い」と感じる（ライダーは待てない） |
| 脈動以外の常時アニメーションは禁止 | バッテリー消費とライダーの注意散漫を防ぐ |

## 8. Haptics（触覚フィードバック）

画面の中だけで完結するフィードバックは、グローブ越しのライダーには届かない。振動は届く。

### 8-1. フィードバックレベル

| レベル | Expo API | 使用場面 |
|--------|----------|----------|
| **Heavy** | `Haptics.impactAsync(Heavy)` | 足跡を刻んだ瞬間（到着確認・「ここに停めた」） |
| **Medium** | `Haptics.impactAsync(Medium)` | CTA タップ確認（案内開始・写真撮影完了） |
| **Light** | `Haptics.impactAsync(Light)` | リスト選択・チップ切り替え |
| **Success** | `Haptics.notificationAsync(Success)` | 同期完了・足跡が誰かの役に立った通知 |
| **Warning** | `Haptics.notificationAsync(Warning)` | 停められなかった記録時 |
| **Selection** | `Haptics.selectionAsync()` | スクロールのスナップ・ピッカー操作 |

### 8-2. Haptics ルール

| ルール | 説明 |
|--------|------|
| 足跡を刻む行為には必ず Heavy を伴わせる | 存在証明の瞬間を身体に刻む。視覚だけでは弱い |
| 連続振動は禁止 | リストスクロール中に毎行振動させない。ライダーの手が疲れる |
| 設定で OFF にできること | `expo-haptics` が使えない端末でも動作すること（フォールバック: 何もしない） |
| 振動 + 視覚を常にペアにする | Haptics だけに頼らない。振動を感じない環境（バイク走行中の振動ノイズ）もある |
| 地図操作中は Haptics なし | ピンチ・パン操作に振動が混ざると邪魔 |

## 9. 光環境とコントラスト

ダークモード専用だが、ライダーは真夏の直射日光下でもスマホを見る。

### 9-1. 環境別コントラスト基準

| 環境 | 条件 | 最低コントラスト比 | 対応 |
|------|------|-------------------|------|
| 夜間・トンネル内 | 暗所、瞳孔開き気味 | 7:1（WCAG AAA） | 現在のパレットで達成済み |
| 日中・日陰 | 標準的な屋外 | 4.5:1（WCAG AA） | 現在のパレットで達成済み |
| 直射日光下 | 画面の視認性が大幅低下 | **テキスト・CTA が判別可能** | 下記ルール参照 |

### 9-2. 直射日光対策ルール

| ルール | 説明 |
|--------|------|
| `sub`（#8E8E93）を単独の情報伝達に使わない | 直射日光下で #8E8E93 on #000000 は読めない。重要情報は `text`（#F2F2F7）を使う |
| CTA ボタンは塗りつぶし（filled）必須 | アウトラインボタンは日光下で消える。`accent` 背景 + 白テキストを死守 |
| アイコンは塗りつぶし（filled）優先 | アウトラインアイコンは細線が日光で飛ぶ。Ionicons の `-outline` より `-sharp` や filled を優先 |
| 地図ピンは白縁取り付き | 明るい地図タイル上でピンが溶けないよう、2px 白ストロークを付与 |
| 温度カラーの最低明度を確保 | cold（#48484A）は日光下で背景と区別しにくい。将来的に日中モードのピンサイズ拡大を検討 |

### 9-3. 検証方法

- 実機を屋外に持ち出して確認する（シミュレータでは再現不可能）
- iOS: 設定 → アクセシビリティ → ディスプレイ → 「明るさの自動調節」ON で最大輝度テスト
- 「信号待ちの3秒で読めるか」を基準にする。じっくり読む時間はない

## 10. 写真表示

ライダーが撮る写真は「作品」ではない。暗い立体駐車場、逆光の看板、手ブレ — 品質はバラバラ。それを前提に設計する。

### 10-1. サムネイル

| 項目 | 値 |
|------|-----|
| アスペクト比 | 4:3（横長） |
| 最小サイズ | 80 × 60 pt |
| カード内サイズ | 幅いっぱい、高さ 180pt |
| 角丸 | `BorderRadius.sm`（8） |
| 背景（読み込み中） | `cardElevated`（#2C2C2E） |
| 複数枚表示 | 横並び最大3枚。4枚以上は「+N」バッジ |

### 10-2. 写真表示ルール

| ルール | 説明 |
|--------|------|
| プレースホルダーはスケルトン | 灰色ベタ塗りではなく、`cardElevated` のパルスアニメーション（控えめ） |
| 写真なしの状態を恥ずかしくしない | 「写真がありません」ではなく、カメラアイコン + 「📸 最初の1枚を残す」 |
| 暗い写真の補正はしない | ユーザーが撮ったままを表示する。フィルタ・自動補正は加えない |
| 写真をタップで拡大表示 | フルスクリーンビューア。ピンチズーム対応。背景は `bg`（#000000） |
| 写真の上にテキストを載せない | 読めない。キャプション（看板/入口/その他タグ）は写真の下に配置 |
| 読み込み失敗時 | 壊れた画像アイコンではなく、`cardElevated` 背景 + カメラアイコン（グレー）|

### 10-3. 写真タグ表示

| タグ | 表示 | 色 |
|------|------|-----|
| 看板 | 「看板」チップ | `accent`（#FF6B00） |
| 入口 | 「入口」チップ | `blue`（#0A84FF） |
| その他 | 表示なし | — |

チップは写真サムネイルの左下に重ねず、サムネイル下部にインラインで配置する。

## 11. コピーライティング（UIテキスト）

| ルール | 説明 |
|--------|------|
| 「報告」と言わない | 「足跡」「メモ」を使う |
| 「ユーザー」と言わない | 「ライダー」「あなた」を使う |
| 「投稿」と言わない | 「残す」「刻む」を使う |
| 敬体（です・ます）で統一 | ただしボタンラベルは体言止め（「案内開始」「足跡を残す」） |
| エラー文は原因 + 次のアクション | 「通信に失敗しました」→「通信に失敗しました。電波の良い場所で再度お試しください」 |
| 空状態は前向きに | 「データがありません」→「まだ足跡がありません。最初の一歩を刻みましょう」 |

## 12. 禁止パターン

以下は Moto-Logos の世界観・ユースケースと矛盾するため、一切使用しない。

| 禁止 | 理由 |
|------|------|
| 星レビュー・5段階評価 | 駐車場は評価するものではない |
| ランキング・リーダーボード | 存在に序列はない |
| ポイント・バッジ・レベル | 報酬で動機づけする思想と矛盾 |
| 横スクロールカルーセル | 片手・グローブ操作に不向き |
| プルダウンセレクト（3項目以上） | タップ回数が増える。ラジオ or チップで |
| モーダル on モーダル | シートの上にモーダルを重ねない |
| ライトモード | ヘルメット越し・夜間を考慮し、ダークのみ |
| トースト通知の多用 | 走行中に一瞬で消える通知は意味がない |
| 全角英数字 | 「１２３」「ＡＢＣ」は禁止。常に半角 |
| `letterSpacing` を本文に適用 | 和文の字間が崩壊する |

## 13. バリデーション・チェックリスト

UI を生成・修正した後、以下を必ず確認する。

### テキスト品質
- [ ] 和文本文の `lineHeight` が fontSize × 1.6 以上か
- [ ] 見出しの `lineHeight` が fontSize × 1.3 以上か
- [ ] `letterSpacing` が本文で 0 になっているか
- [ ] 3行以上に折り返される和文テキストの可読性を実機確認したか
- [ ] 混植テキスト（スポット名、車種名）の表示崩れがないか

### タッチ品質
- [ ] すべてのタップ可能要素が 52pt 以上か
- [ ] 主要 CTA が 64pt 以上か
- [ ] 隣接するタップ領域間に 8pt 以上のギャップがあるか
- [ ] 片手（親指）で主要操作が完結するか
