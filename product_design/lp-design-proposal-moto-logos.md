# Moto-Logos LP デザイン提案書

**作成:** Anna (Product Design)
**日付:** 2026-04-12
**ステータス:** 初版 / レビュー待ち
**対象:** Vite + React / Firebase Hosting
**ビューポート基準:** 375px (iPhone SE) をモバイルファースト起点とし、768px / 1280px でブレイクポイント

## 0. 設計思想 — 「企業サイト」ではなく「ライダーの地図」

このLPは「サービスの説明」ではない。
ライダーが走り終えてスマホを見たとき、「ああ、これは俺たちのためのものだ」と直感する場所をつくる。

**設計原則:**
- 暗闘の中に灯るオレンジ。夜道のヘッドライトのような視線誘導
- 手袋を外したばかりの指で触れても迷わないタップ領域
- 情報量は多く、しかし「読ませる」のではなく「流れてくる」構成
- コピーはすべて「語りかけ」。一人称ではなく二人称で、ライダー自身の言葉に聞こえるように

**カラーシステム（アプリ準拠）:**

| トークン | 値 | 用途 |
|----------|-----|------|
| `--bg` | `#0D0D0D` | ページ背景 |
| `--surface` | `#1A1A1A` | カード・セクション背景 |
| `--card` | `#242424` | 内部カード・FAQ |
| `--accent` | `#FF6B00` | CTA・強調・ホバー |
| `--accent-light` | `#FF8C33` | グラデーション終点 |
| `--text` | `#F5F5F5` | 本文 |
| `--text-secondary` | `#A0A0A0` | 補助テキスト |
| `--fresh-blue` | `#0A84FF` | 鮮度バッジ（新しい） |
| `--fresh-yellow` | `#FF9F0A` | 鮮度バッジ（やや古い） |
| `--fresh-red` | `#FF453A` | 鮮度バッジ（古い） |
| `--success` | `#30D158` | ポジティブ指標 |
| `--purple` | `#BF5AF2` | ユーザー投稿マーカー |

**タイポグラフィ:**
- 見出し: `Noto Sans JP` Bold / 32-48px（モバイル）、48-64px（デスクトップ）
- 本文: `Noto Sans JP` Regular / 15-16px、line-height 1.8
- 数値（カウンター等）: `Inter` または `DIN` Bold / Tabular Nums

### コピー案

**メインヘッドライン（候補3案、A/Bテスト想定）:**

A. **「駐輪場、見つからない」を終わらせにきた。**
B. **止める場所がないなら、ライダーでつくればいい。**
C. **バイク乗りが、バイク乗りのために。**

**サブコピー:**
> 一人の発見を、全ライダーの安心に。
> ライダー同士がリアルタイムで駐輪場を共有する、日本初のコミュニティマップ。

### ビジュアル演出

- **背景:** `#0D0D0D` 単色。余計なグラデーションは入れない。暗闘の中にコンテンツだけが浮かぶ

- **スクロール誘導:** 画面最下部にシェブロンアイコン（`opacity` が `0.3→0.8` で呼吸）

## 2. ペインセクション — 「その駐輪場情報、何年前のもの？」

### 設計意図

ここが最大の「介入ポイント」。ライダーなら誰もが経験した苛立ちを言語化し、「分かる」という共感から入って、既存サービスへの不信感を自然に想起させる。直接的な競合批判はしないが、「3年前のデータ」というフレーズで暗黙的に差別化する。

### レイアウト

```
┌─────────────────────────────────┐
│                                  │
│  セクション見出し:                │
│  「着いてみたら、閉鎖されていた。」│  ← 28px Bold / text
│                                  │
│  ┌─────────┐  ┌─────────┐       │
│  │ 😤       │  │ 😤       │       │  ← ペインカード x3（縦積み）
│  │ ナビアプリ │  │ 情報が古い │       │
│  │ に駐輪場   │  │ 行ったら   │       │
│  │ が載って   │  │ 閉鎖済み   │       │
│  │ いない     │  │           │       │
│  └─────────┘  └─────────┘       │
│  ┌─────────┐                    │
│  │ 😤       │                    │
│  │ 大型不可？│                    │
│  │ 着くまで  │                    │
│  │ 分からない│                    │
│  └─────────┘                    │
│                                  │
│  「最終更新: 3年前」              │  ← 鮮度バッジ（赤）のデモ
│  ← こんな情報、信じられますか？    │     アプリ実装と同じUI
│                                  │
└─────────────────────────────────┘
```

### コピー案

**セクション見出し:**
> 着いてみたら、閉鎖されていた。

**ペインカード（3枚）:**

1. **「ナビに載っていない」**
   バイク駐輪場は Google Maps にもカーナビにも、ほとんど出てこない。車は停められるのに、バイクだけが路上を彷徨う。

2. **「情報が古い。行ったら閉鎖」**
   ネットの駐輪場情報、最後に更新されたのはいつ？ 3年前のデータで「あります」と言われても、着いたら工事中。

3. **「大型お断り。着くまで分からない」**
   排気量制限は現地に行かないと分からない。400ccで停められるか、原付専用か。賭けで走るしかない。

**暗示的差別化テキスト（カード下部、小さく）:**

> [赤い鮮度バッジ] 最終更新: 3年前
> この情報で、安心して走れますか？

### ビジュアル演出

- **ペインカード:** `surface (#1A1A1A)` 背景、左に赤い縦ライン（4px、`#FF453A`）。苛立ちを視覚化
- **鮮度バッジデモ:** アプリ内実装と完全に同じビジュアル。赤バッジ（`#FF453A`背景18%透過、`#FF453A`ボーダー）に `warning` アイコン + 「3年前」テキスト。これがLP上でインタラクティブに動く（後述のSection 3で青バッジと対比）
- **スクロールトリガー:** カードはビューポート侵入時に左から `translateX(-30px)→0` でスライドイン。0.15秒ずつディレイ
- **背景:** `#0D0D0D` のまま。余白を大きく取り、圧迫感を出さない

### レスポンシブ

- モバイル: カード縦積み、フル幅
- タブレット+: 3カード横並び（`grid: repeat(3, 1fr)`）

## 3. コアバリュー — 3つの武器

### 設計意図

ペインの直後に「解決策」を提示。ただし「機能紹介」にならないよう、各バリューはライダーの体験として語る。各カードにアプリの実際のスクリーンショット（アニメーション付き）を配置し、「これが動くんだ」という実感を持たせる。

### レイアウト

```
┌─────────────────────────────────────────┐
│                                          │
│  セクション見出し:                         │
│  ライダーが、ライダーのためにつくった。      │
│                                          │
│  ═══ バリュー 1 ═══════════════════════   │
│  ┌──────────────────────────────────┐    │
│  │  [アプリ画面: ラジアルメニュー]      │    │
│  │   親指1本操作のデモ動画             │    │
│  │   3-5秒ループ / autoplay / muted   │    │
│  └──────────────────────────────────┘    │
│                                          │
│  「グローブのまま、0.5秒で報告」           │  ← 20px Bold
│                                          │
│  信号待ちの数秒で、駐輪場の「今」を         │  ← 15px / textSecondary
│  共有できる。親指1本のラジアルメニュー。     │
│  グローブを外す必要すらない。               │
│                                          │
│  ═══ バリュー 2 ═══════════════════════   │
│  ┌──────────────────────────────────┐    │
│  │  [インタラクティブ鮮度バッジデモ]     │    │  ← LP最大の武器
│  │                                    │    │
│  │  🔵 最新   → 🟡 2ヶ月前 → 🔴 1年前  │    │
│  │    スライダーで時間経過をシミュレート   │    │
│  │    バッジの色とアイコンがリアルタイム変化│    │
│  └──────────────────────────────────┘    │
│                                          │
│  「データに鮮度がある。一目で分かる。」     │  ← 20px Bold
│                                          │
│  すべてのスポットに鮮度バッジ。             │
│  青は新鮮、黄色は確認推奨、赤は要注意。     │
│  いつ誰が確認したか、すべて透明。           │
│                                          │
│  ═══ バリュー 3 ═══════════════════════   │
│  ┌──────────────────────────────────┐    │
│  │  [マップ画面: ピンが増殖する動画]     │    │
│  │   タイムラプス風にピンが増えていく     │    │
│  │   最初は数個 → 画面いっぱいに         │    │
│  └──────────────────────────────────┘    │
│                                          │
│  「走れば走るほど、地図が育つ。」           │  ← 20px Bold
│                                          │
│  あなたが発見したスポットが地図に刻まれ、    │
│  次に走るライダーを助ける。                │
│  一人の報告が、全員の安心になる。           │
│                                          │
└─────────────────────────────────────────┘
```

### コピー案（各バリュー）

**バリュー1: 片手操作**
- 見出し: **グローブのまま、0.5秒で報告。**
- 本文: 信号待ちの数秒で、駐輪場の「今」を共有できる。親指1本のラジアルメニュー。グローブを外す手間は、もういらない。
- アプリ内対応: `RadialMenu` コンポーネント（長押しで扇形展開、ドラッグ選択）

**バリュー2: 鮮度バッジ**
- 見出し: **データに賞味期限がある。一目で分かる。**
- 本文: すべてのスポットに「鮮度バッジ」。青は直近30日、黄色は1-3ヶ月、赤は3ヶ月以上未確認。その情報がいつのものか、もう迷わない。
- アプリ内対応: `FreshnessBadge` コンポーネント（`checkmark-circle`/`alert-circle`/`warning` アイコン + 色分け）

**バリュー3: 集合知マップ**
- 見出し: **走れば走るほど、地図が育つ。**
- 本文: あなたが見つけたスポットが地図に刻まれ、次に走るライダーの道しるべになる。使うだけじゃない。つくるアプリ。
- アプリ内対応: Firestoreリアルタイム同期 + クラスタリングマップ

### ビジュアル演出

**バリュー1 - グローブ操作デモ:**
- 実際にグローブをはめた手でiPhoneを操作する動画（3-5秒ループ）
- `<video autoplay muted loop playsinline>` でモバイル自動再生
- 代替: WebPアニメーション（動画より軽量）
- ビデオ外枠: `border-radius: 24px`、`box-shadow: 0 8px 32px rgba(255,107,0,0.15)`

**バリュー2 - インタラクティブ鮮度バッジ（LP最大の独自武器）:**
- LP上にスライダー（range input）を設置。ドラッグすると：
  - 0日→「最新」（青バッジ、`#0A84FF`、`checkmark-circle`アイコン）
  - 45日→「2ヶ月前」（黄バッジ、`#FF9F0A`、`alert-circle`アイコン）
  - 180日→「6ヶ月前」（赤バッジ、`#FF453A`、`warning`アイコン）
  - 1095日→「3年前」（赤バッジ、ドクロ演出 + テキスト「この情報、信じますか？」）
- バッジUIはアプリ内 `FreshnessBadge` と完全に同一のビジュアル（pill shape、透過背景、アイコン+テキスト）
- スライダーのトラック色もバッジに連動（青→黄→赤のグラデーション遷移）
- **実装:** React state + CSS transitions。Intersection Observerでスクロール到達時に自動で一度動く。その後はユーザー操作可能

**バリュー3 - マップ増殖アニメーション:**
- Lottieまたはキャンバスアニメーション。暗い地図上にピンが1つずつ出現（pop animation）
- 最初の3秒で5個→10秒で50個→「あなたのスポットもここに」テキスト出現
- 代替: 静止画3枚のカルーセル（3ヶ月前 / 1ヶ月前 / 今日のマップ比較）

### レスポンシブ

- モバイル: 画像→テキストの縦積み（全バリュー共通）
- デスクトップ: バリュー1は左画像/右テキスト、バリュー2は右画像/左テキスト（ジグザグ配置）。バリュー3はフル幅アニメーション

## 4. ライブ統計カウンター

### 設計意図

「このアプリは生きている」ことを数字で証明する。静的な数字ではなく、Firestoreからリアルタイムに取得するカウンターで、ページを開いている間にも数字が増える可能性があることを示す。

### レイアウト

```
┌──────────────────────────────────────┐
│  ─── 仲間が動いている ───             │  ← 小さなセクション見出し
│                                       │
│  ┌──────┐  ┌──────┐  ┌──────┐        │
│  │ 1,247 │  │  312  │  │  89%  │        │
│  │共有済み │  │今週の  │  │鮮度   │        │
│  │スポット │  │更新   │  │30日内 │        │
│  └──────┘  └──────┘  └──────┘        │
│                                       │
│  ── 直近のアクティビティ ──            │
│  🟣 渋谷駅前に新スポット追加 (2分前)    │  ← Firestoreリアルタイム
│  🔵 新宿西口の情報を確認 (8分前)       │     フィード
│  🟢 池袋東口「空きあり」報告 (15分前)   │
│                                       │
└──────────────────────────────────────┘
```

### コピー案

**セクション見出し:** 今この瞬間も、仲間が走っている。

**カウンター項目:**
1. **共有スポット数** — Firestoreの `spots` コレクション総数
2. **今週の更新数** — 直近7日間のステータス報告数
3. **鮮度カバー率** — 30日以内に確認されたスポットの割合（%）

**ライブフィード見出し:** 直近のアクティビティ

### ビジュアル演出

- **カウンター:** 数値は `countUp.js` 的なアニメーション。スクロール到達時に0から現在値まで1.5秒で上昇
- **数字フォント:** `Inter` Bold Tabular Nums、`48px`、`#F5F5F5`。下のラベルは `14px`、`#A0A0A0`
- **カウンター背景:** `#1A1A1A` カード、`border: 1px solid #333333`、`border-radius: 16px`
- **ライブフィード:** Firestore `onSnapshot` でリアルタイム購読。新しいエントリが上から `slideDown` で追加される
  - 各行の左にカラードット（ユーザー投稿=紫 `#BF5AF2`、確認報告=青 `#0A84FF`、空き報告=緑 `#30D158`）
  - タイムスタンプは相対表示（「2分前」「8分前」）
- **フォールバック:** Firestore接続不可時は、直近のスナップショットデータを表示（キャッシュ）

### レスポンシブ

- モバイル: カウンター3つは横並び（各33%）、フィードはフル幅
- デスクトップ: カウンター3つを大きく表示 + 右にフィード

## 5. ソーシャルプルーフ — ライダーの声

### 設計意図

日本市場では「自分と似た立場の人の声」が極めて強い。年齢・バイク種・利用シーンが異なる3-5名のライダーを起用し、「どの層にも刺さる」配置にする。写真はバイク+ライダーの実写が必須。スタジオ写真ではなく、ツーリング先やガレージでの自然体。

### レイアウト

```
┌──────────────────────────────────────┐
│                                       │
│  走った仲間の声。                      │  ← セクション見出し
│                                       │
│  ┌──────────────────────────────┐     │
│  │  "都内はほんとに停める場所なくて   │     │  ← テスティモニアルカード
│  │   毎回コインPで罰金ビクビクだった。│     │
│  │   これ入れてから駐輪場ハズレが     │     │
│  │   マジでなくなった。"              │     │
│  │                                 │     │
│  │  [写真] 田中さん (28)            │     │
│  │  CB400SF / 通勤ライダー          │     │
│  └──────────────────────────────┘     │
│                                       │
│  ← スワイプで次へ →                   │  ← カルーセル / 横スクロール
│                                       │
└──────────────────────────────────────┘
```

### コピー案（テスティモニアル 5名分）

**1. 通勤ライダー（都市部・ミドルクラス）**
> "都内は本当に停める場所がなくて、毎回コインPで罰金ビクビクだった。Moto-Logos入れてから駐輪場のハズレがなくなった。鮮度バッジ、マジで神。"
> — 田中さん (28) / CB400SF / 都内通勤

**2. ツーリングライダー（休日・大型）**
> "知らない土地でバイク停める場所を探す時間が、ツーリングの中で一番嫌いだった。今は出発前にチェックするだけ。しかも他のライダーが最近確認した情報だから安心。"
> — 鈴木さん (35) / Ninja 1000 / 週末ツーリング

**3. 原付ライダー（学生・エントリー）**
> "原付だから大丈夫だろうと思って行ったら「二輪不可」。何回泣かされたか。排気量フィルターがあるのが本当にありがたい。"
> — 佐藤さん (22) / PCX125 / 大学通学

**4. 女性ライダー**
> "一人で走る時、知らない場所で路駐は不安だった。誰かが「空きあり」って報告してくれてると、それだけで安心して走れる。"

## 7. スティッキーCTA（画面追従）

### 設計意図

日本市場標準のスティッキーCTA。ただし「常に見えている」のではなく、ヒーローのCTAがビューポートから消えたタイミングで下からスライドインし、フッターCTA到達時に消える。「鬱陶しい」ではなく「ちょうどいい」タイミング設計。

### レイアウト

```
┌──────────────────────────────────────┐
│  Moto-Logos を無料で始める     [入手]  │  ← 高さ60px、画面下部固定
│                                       │     背景: rgba(13,13,13,0.95)
│                                       │     backdrop-filter: blur(20px)
└──────────────────────────────────────┘
```

### 仕様

- **出現条件:** ヒーローセクションのCTAボタンが `IntersectionObserver` でビューポート外になったとき
- **消失条件:** フッターCTAセクションがビューポートに入ったとき
- **アニメーション:** `translateY(100%)→0`、`0.3s ease-out`
- **背景:** `rgba(13,13,13,0.95)` + `backdrop-filter: blur(20px)`。セクション内容がうっすら透ける
- **ボタン:** `#FF6B00` 背景、`border-radius: 12px`、`height: 44px`
- **テキスト:** 左に「Moto-Logos を無料で始める」(`#F5F5F5`, 14px)、右に「入手」ボタン
- **セーフエリア対応:** iOS の home indicator 領域に `padding-bottom: env(safe-area-inset-bottom)` を適用

### レスポンシブ

- モバイルのみ表示（768px以上では非表示。デスクトップでは各セクションCTAで十分）

## 8. FAQ セクション

### 設計意図

コンバージョンの最終障壁を取り除く。日本のユーザーは「無料？本当に？」「個人情報は？」という不安を持ちやすい。アコーディオン形式で、クリックするまで回答を隠すことで、ページの圧迫感を抑える。

│                                       │

### コピー案（Q&A 5問）

**Q1: 本当に無料ですか？**
> はい、完全無料です。広告もありません。Moto-Logosはライダーコミュニティで成り立つアプリです。将来的にプレミアム機能を検討する可能性はありますが、基本機能はずっと無料です。

**Q2: 位置情報は常に取得されますか？**
> いいえ。位置情報はアプリを開いている間のみ使用し、バックグラウンドでの追跡は一切行いません。位置情報はスポット検索と報告にのみ使用されます。

**Q3: どんなバイクでも使えますか？**
> 原付（50cc）から大型（排気量無制限）まで対応しています。排気量フィルターを設定すれば、あなたのバイクで停められるスポットだけが表示されます。

**Q4: スポット情報の正確性はどう担保していますか？**
> すべてのスポットに「鮮度バッジ」があり、最後に確認された日時が一目で分かります。さらに、複数のライダーによるステータス報告（空きあり / 満車 / 閉鎖）でリアルタイムに情報が更新されます。古い情報は自動的に赤バッジで警告されます。

**Q5: 自分が投稿した情報は他の人に見られますか？**
> スポットの位置情報やステータスは全ライダーに共有されますが、投稿者の個人情報（メールアドレス、位置履歴など）が他のユーザーに公開されることはありません。ニックネームのみ表示されます。

### ビジュアル演出

- **アコーディオン:** `#1A1A1A` 背景、`border: 1px solid #333`
- **展開アニメーション:** `max-height: 0→auto`（`transition: max-height 0.3s ease-out`）+ シェブロン回転（`transform: rotate(0→180deg)`）
- **シェブロンアイコン:** `#FF6B00`、展開時に色が `#F5F5F5` に変化
- **Q テキスト:** `16px Bold #F5F5F5`
- **A テキスト:** `15px Regular #A0A0A0`、`line-height: 1.8`

### レスポンシブ

- 全デバイス共通レイアウト（フル幅アコーディオン）
- デスクトップでは最大幅 `720px` でセンタリング

## 10. 全体設計: アニメーション & インタラクション仕様

### スクロール連動アニメーション

| セクション | トリガー | アニメーション | 時間 |
|-----------|----------|---------------|------|
| ヒーロー | ページ読込 | テキスト行ごとフェードイン + アプリモック浮遊 | 0.3s/行 |
| ペイン | 50%表示 | カード左→右スライドイン | 0.4s, 0.15s delay |
| バリュー1 | 50%表示 | 動画自動再生 + テキストフェードイン | 0.5s |
| バリュー2 | 50%表示 | 鮮度スライダー自動1往復 | 2s |
| バリュー3 | 50%表示 | ピン増殖アニメ開始 | 3s |
| カウンター | 30%表示 | 数値カウントアップ | 1.5s |
| ライブフィード | 30%表示 | 項目が上からスライドイン | 0.3s/項目 |
| テスティモニアル | 50%表示 | カードフェードイン | 0.5s |
| ステップ | 30%表示 | プログレスバー進行 | スクロール連動 |
| ファイナルCTA | 50%表示 | ボタンパルス開始 | ループ |

### パフォーマンス指針

- `IntersectionObserver` で遅延読み込み。アニメーションはビューポート外では停止
- 動画は `<video>` タグ + `loading="lazy"`。初期表示はポスター画像
- 鮮度バッジデモはCSS Transitionsのみ。JSアニメーションライブラリ不要
- ライブフィードはFirestore `onSnapshot` だが、1分間隔のポーリングにフォールバック
- **目標:** Lighthouse Performance 90+、LCP 2.5秒以内、CLS 0.1以下

### アクセシビリティ

- すべてのインタラクティブ要素に `aria-label`
- 動画には `<track>` で字幕（操作説明テキスト）
- `prefers-reduced-motion` でアニメーション無効化対応
- コントラスト比: `#F5F5F5` on `#0D0D0D` = 18.1:1（AAA適合）
- スティッキーCTAは `role="complementary"` + `aria-live="polite"`

## 12. 「コミュニティ」と「鮮度」の視覚言語

このLPの全体を通じて、2つの概念を繰り返し視覚的に伝える。

### コミュニティ = 「ピンが増える」「声が聞こえる」「数字が動く」

- **ヒーロー:** マップ上のピンが脈動（一人の発見）
- **バリュー3:** ピンが増殖するアニメーション（集合知）
- **カウンター:** リアルタイムで増える数字（仲間が動いている）
- **ライブフィード:** 実際のアクティビティ（今この瞬間の仲間）
- **テスティモニアル:** 顔と声（コミュニティは抽象概念ではなく人）
- **カラーコード:** ユーザー投稿には常に `#BF5AF2`（紫）を使用。アプリ内のマーカー色と統一

### 鮮度 = 「色が変わる」「時間が見える」「古さが警告になる」

- **ペインセクション:** 赤バッジ「3年前」で既存の古さを糾弾
- **バリュー2:** インタラクティブスライダーで色遷移を体験
- **FAQ Q4:** 鮮度バッジの仕組みを文章で補強
- **カラーコード:** LP全体で一貫して `青=新鮮` / `黄=注意` / `赤=危険` のセマンティックカラーを維持。アプリ内の `FreshnessBadge` と完全同一

### 繰り返しのリズム

LP全体で「問題提起 → 解決策 → 証拠 → 行動喚起」のリズムを2回繰り返す:

1. **第1サイクル:** ペイン → バリュー → カウンター（論理的説得）
2. **第2サイクル:** テスティモニアル → ステップ → ファイナルCTA（感情的説得）

FAQ は両サイクルの間に挿入し、理性的な障壁を処理する。
