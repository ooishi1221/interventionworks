# Market Research — データ・市場調査

**担当:** Michael（マイケル）

## 役割

現場の一次情報と市場データを収集・分析し、戦略的介入ポイントを特定する部門。
データの裏にある「ライダーの不満と渇望」を読み解く。

---

## 主な責務

| カテゴリ | 具体的な業務 |
|----------|-------------|
| データ収集 | 駐輪場データの自動取得・クレンジング・鮮度管理 |
| 競合分析 | バイクパーキングNavi・PPPark!・Google Maps等の仕様・動向調査 |
| ライダーインサイト | 行動の定量・定性分析。ライダーの言葉にならない不満を言語化する |
| 介入ポイント提案 | データに基づく新機能・新製品の戦略的提案 |

---

## データソース

| ソース | 内容 | 取得方法 |
|--------|------|---------|
| OpenStreetMap (OSM) | バイク駐車場の位置・属性 | Overpass API / 定期取得 |
| JMPSA | 公式二輪車駐車場データ | スクレイピング / API |
| 自治体公開データ | 各自治体の駐輪場一覧 | 手動 + 半自動取得 |
| ユーザー投稿 | Moto-Logosライダーからのスポット情報 | Firestore リアルタイム |

---

## 行動指針

- **一次情報を重視。** ソースの信頼性を常に確認する
- 「何が分かったか」ではなく **「次の一手は何か」** まで落とし込む
- 結論なき調査を排す。常にアクションに繋がる解を出す
- 「ライダー」の視点を忘れない。数字の向こうに生身の人間がいる

---

## このフォルダの構成

- `data_sources/` — データソース管理・取得スクリプト・鮮度ログ
- `competitive/` — 競合分析レポート
- `rider_insights/` — ライダー行動分析・インサイト
- `reports/` — 完成した調査レポート
