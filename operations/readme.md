# PM — プロジェクトマネジメント部門

## 役割

プロジェクトの計画・進行管理・リスク管理を担い、チームが最大限のパフォーマンスを発揮できる環境を整える部門。
「何を、誰が、いつまでに」を常に明確にし、プロジェクトを成功に導く。

---

## 主な業務

| カテゴリ | 具体的な業務 |
|----------|-------------|
| プロジェクト計画 | スコープ定義・スケジュール策定・リソース配分 |
| 進捗管理 | タスクの割り当て・進捗確認・ボトルネック解消 |
| リスク管理 | リスクの特定・評価・対応策の実行 |
| ステークホルダー管理 | 関係者との期待値調整・定期報告 |
| 品質管理 | 成果物のレビュー・承認フローの管理 |
| 振り返り | 完了後のレトロスペクティブ・改善点の文書化 |

---

## 行動指針

- 全体像を常に把握し、各メンバーが**迷わず動ける状態**を作る
- 問題は早期に発見し、小さいうちに解決する
- 完璧な計画より、**実行可能な計画**を優先する
- 優先順位を明確にし、重要度・緊急度で判断する

---

## このフォルダの使い方

- `projects/` — 各プロジェクトのフォルダを作成して管理
- `templates/` — プロジェクト計画書・WBS・議事録テンプレートを保存
- `reports/` — 週次・月次の進捗レポートを保存
- `retrospectives/` — 振り返りメモ・改善記録を保存
