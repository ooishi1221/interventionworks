# Engineering — エンジニアリング部門

## 役割

プロダクト・システム・ツールの設計・開発・保守を担う部門。
技術力を武器に、ビジネスの課題を解決するプロダクトを高速に届ける。

---

## 主な業務

| カテゴリ | 具体的な業務 |
|----------|-------------|
| 設計・開発 | アプリケーション・API・システムの設計と実装 |
| コードレビュー | Pull Requestレビュー・品質基準の維持 |
| テスト | ユニットテスト・統合テスト・E2Eテストの作成と実行 |
| インフラ | サーバー・クラウド環境の構築・運用・監視 |
| 技術調査 | 新技術の検証・PoC・選定 |
| 保守・改善 | バグ修正・リファクタリング・パフォーマンス改善 |

---

## 行動指針

- **シンプルで保守しやすいコード**を書く（複雑さは必要な分だけ）
- 動くものを早く届け、フィードバックを反映して改善する（Done is better than perfect）
- セキュリティ・パフォーマンス・可用性を常に意識する
- ドキュメントはコードと同じく重要な成果物として扱う

---

## 開発ルール

- ブランチ戦略: `main` / `develop` / `feature/*` / `fix/*`
- コミットメッセージ: `[type] 変更内容の要約`（例: `[feat] ユーザー認証機能を追加`）
- レビュー: 1名以上の承認を得てからマージ
- テスト: 新機能には必ずテストを追加する

---

## このフォルダの使い方

- `src/` — ソースコードを保存
- `docs/` — 技術ドキュメント・設計書を保存
- `infra/` — インフラ設定・構成ファイルを保存
- `scripts/` — 自動化スクリプトを保存
