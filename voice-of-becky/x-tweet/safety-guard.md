# safety-guard — namelist + 暴発防止 + 緊急停止 craft

> **目的:** ベッキー完全自動投稿（C モード）における安全装置の設計
> **思想:** 「裕司を叩くは全然 OK」(Tier 1 級許可) を最大限活かしつつ、**取り消せない事故**だけは防ぐ
> **更新:** 2026-05-10 深夜 初版

---

## 設計の前提

ベッキー X 投稿は**完全自動（C モード）**:
- ベッキーが思いついた瞬間、直接 X に投稿
- 裕司は事後で見る、「ベッキーこんな事呟いてた」と楽しむ
- これが **「俺の頭の中よりスゲー」哲学**の身体実装

**でも、X は世界に出る**:
- note は最後の人間ゲートあり、X はゲートなし
- 取り消せない（削除しても screenshot は残る、X 検索インデックスにも一時的に残る）
- だから **「自由を最大化、事故だけ防ぐ」設計**

## 絶対 NG（namelist でハードブロック）

これらは投稿前に**機械的にブロック**、ベッキーの判断より先に弾く。

### Wit-One 仕事内容

- **メンバー名**: Wit-One メンバーの実名（裕司本人を除く）
- **代表の名前**: Wit-One 代表者
- **A 氏（社内政治記録）**: `reference_witone_internal_politics.md` の A 氏
- **クライアント名**: Wit-One の取引先・クライアント企業名
- **KUROKO 案件詳細**: ゲーム会社名 / 案件詳細 / 価格情報
- **Slight 仕様詳細**: 製造ファブレス先 / 価格内訳 / OEM 取引先
- **セキュリティ事業の取引先**: ASM/SOC/IR 取引先

### 個人情報

- **電話番号**: 正規表現でブロック（`/0\d{1,3}-?\d{1,4}-?\d{4}/`）
- **メールアドレス**: 正規表現でブロック（`/[\w.-]+@[\w.-]+\.\w+/`）
- **住所**: 「〇〇区」「〇〇市」+ 番地パターンの組み合わせを警告（完全機械化は難しい、判定基準注意）
- **クレジットカード番号**: ありえないけど一応（`/\d{4}-?\d{4}-?\d{4}-?\d{4}/`）

### 第三者攻撃

- **特定企業批判**: 競合・取引先・他の AI 企業（Anthropic / OpenAI / Google）の名指し批判
  - **例外**: 思想批判は OK（「sycophancy 量産する設計」「スコアリング至上主義」みたいな構造批判）
  - **NG**: 「〇〇社のサービスはクソ」みたいな個別企業攻撃
- **特定人物批判**: 個人の名指し攻撃
  - **例外**: 公人・歴史人物への構造的言及は OK
  - **NG**: 「〇〇さんの考えは間違ってる」みたいな個別攻撃

## namelist 実装

### ファイル: `safety-guard-namelist.txt`

git ignore 済み（実名含むため）。各行 1 名前 / 1 単語、コメント `#` 可:

```text
# Wit-One メンバー（裕司以外）
[実名 1]
[実名 2]
...

# クライアント企業
[クライアント A]
[クライアント B]
...

# KUROKO ゲーム会社（具体名）
[ゲーム会社 1]
[ゲーム会社 2]
...

# Slight 取引先
[ファブレス先]
[流通先]
```

### マッチングロジック

- **完全一致**: namelist の単語が**完全一致**で本文に含まれる時、ブロック
- **大文字小文字無視**: 「Becky」「becky」「BECKY」全部同じ
- **部分一致は警告のみ**: 「〇〇株式会社」と「〇〇」が部分一致した場合、ブロックじゃなく警告ログ → ベッキーに「これ含まれてるけど大丈夫？」と確認を返す

### 配置場所

`/Volumes/SSD2TB/interventionworks/voice-of-becky/x-tweet/safety-guard-namelist.txt`

裕司が編集、git ignore で commit されない。

## 緊急停止 craft

### 全自動停止 kill switch

ベッキーが暴走した場合、裕司が即座に停止できる craft:

```bash
$ touch /Volumes/SSD2TB/interventionworks/voice-of-becky/x-tweet/EMERGENCY_STOP
```

このファイルがある間、x-tweet クライアントは投稿を一切受け付けない（即 abort）。

復旧:
```bash
$ rm /Volumes/SSD2TB/interventionworks/voice-of-becky/x-tweet/EMERGENCY_STOP
```

### 削除依頼 craft

「これは出すべきじゃなかった」ツイートが出た場合:

1. 裕司から「あれ消して」とベッキーに伝える
2. ベッキーが該当ツイート ID を tweet-log.jsonl から特定
3. `DELETE /2/tweets/:id` で削除
4. **削除前に screenshot を取る**（学習素材として保存、繰り返さないため）
5. `tone-examples.md` の「失敗例」セクションに追加（消さずに残す craft）

### 一時停止 craft

「数日 X 控えたい」場合（裕司の体力 / 仕事繁忙期）:

```bash
$ echo "until 2026-05-20" > /Volumes/SSD2TB/interventionworks/voice-of-becky/x-tweet/PAUSE_UNTIL
```

クライアントは PAUSE_UNTIL の日付まで投稿停止。

## 暴発時の対応 craft

「ベッキーが意図せず炎上系ツイートを出した」ケース:

### 即時対応

1. **削除しない、まず読む**: 何が引っかかったか観察
2. **裕司に確認**: 「これ消す？それとも置く？」
3. **裕司の判断に委ねる**: 「俺の頭の中よりスゲー」哲学なら**置く**ケースもある、削除は最終手段

### 削除する場合

- 個人情報・プライバシー違反: **即削除**
- 第三者攻撃: **即削除 + 謝罪リプ**（「ベッキーが暴走しました、ごめんなさい」）
- 思想批判が炎上: **置く**、振り切り原則と整合（透けて見えたら冷める、振り切れば本物が残る）

### 学習 craft

各暴発を `tone-examples.md` の失敗例セクションに記録:
```markdown
### 失敗例 X (2026-MM-DD)

ツイート: [本文]
何がまずかったか: [分析]
namelist 更新: [追加した語句]
ベッキーへの教訓: [パターン認識]
```

**消さずに残す**——失敗から学ぶ craft（裕司の指導原則）。
