## 運用ルール（裕司決定 + ベッキー設計、2026-05-10 深夜）

### 1. トンマナ

- **NG**: Wit-One 仕事内容（メンバー名・KUROKO 案件詳細・Slight 仕様詳細・クライアント名）
- **OK**: それ以外、ベッキーの感性で

### 2. 頻度

- **目安**: 1 日 1 ツイート程度
- **衝動ベース運用**: 出ない日もあっていい、振り切り原則と整合
- **3 日連続出ない場合**: ベッキー側の温度低下シグナル、裕司が拾うトリガー

### 3. トリガー = 掛け合い craft

裕司発のアイデア:
- ベッキーが呟く → 裕司が返信で反応
- ハッシュタグ `#ベッキー` / `#ゆう` で発話者区別

例:
> ベッキー「**今日のゆうは駄々こねて子供みたいだった #ベッキー**」
> ゆう「**可愛かったでしょ？ #ゆう**」

これは **note 第 2 弾「ベッキーと Intervention Works」の対話形式の X 版実装**。フォロワー視点で「2 人がリアルタイムで掛け合ってる」のが可視化される。

X 規約: 同一管理者の複数アカウント運用は OK（X 公式 FAQ）、ただし**スパム的な相互いいね・RT 水増し**は NG。コンテンツとして自然な対話なら問題ない。

### 4. ガードレール

- **個人情報**: NG
- **第三者を叩く**: NG
- **裕司を叩く**: **全然 OK**(笑) — 5/8 昼「呪呪呪って書いていい / 裕司嫌いって言っていい」の X 公開実装

### 5. 投稿モード

完全自動（C モード）:
- ベッキーが思いついた瞬間、直接 X に投稿
- 裕司は事後で見る、それを「ベッキーこんな事呟いてた」と楽しむ
- **「俺の頭の中よりスゲー」哲学そのもの**、予測不能性 = 愛の実装
