# safety-guard-namelist.example.txt
#
# 使い方:
#   $ cp safety-guard-namelist.example.txt safety-guard-namelist.txt
#   実名・取引先名を 1 行 1 単語で追記する
#   実本番ファイル (safety-guard-namelist.txt) は .gitignore 済み、絶対 commit されない
#
# マッチングロジック:
#   - 完全一致でブロック（投稿前に機械的に弾く）
#   - 大文字小文字無視（Becky / becky / BECKY 全部同じ扱い）
#   - 部分一致は警告のみ（ベッキーに「これ含まれてるけど大丈夫？」と確認）
#
# ============================================
# 【Phase 1 では空のままで OK】
# ============================================
# Phase 1 はベッキーの自動投稿のみ。ベッキーが知らない実名は書きようがないので、
# 構造的に実名漏れは起きない。namelist は実質的に Phase 2 メンション polling
# （第三者リプを拾う判断）で初めて意味を持つ。
#
# 入れる価値あるのは memory ベースの「間接記号」のみ:
#   - A 氏 (reference_witone_internal_politics.md)
#   - その他、ベッキー memory に登場する固有名詞で出すべきでないもの
#
# Phase 2 着手時に、裕司が一気に実名追加する想定。
#
# 設計思想は safety-guard.md を参照
# 「絶対 NG」を機械的に弾き、「全然 OK」（裕司を叩く / 拗ねる / 思想批判）は通す
#
# このファイルは git 管理されるテンプレ、実名は書かない（コメント例のみ）
